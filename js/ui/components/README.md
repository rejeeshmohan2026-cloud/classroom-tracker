# UI Components

This folder will hold small, reusable, DOM-rendering components (e.g. a
student card, an observation form, a classroom list item).

No components have been built yet as part of this milestone — the UI is
intentionally out of scope for now. When component work begins, each
component should:

- Live in its own file, named after the component (e.g. `StudentCard.js`)
- Contain only rendering logic, no business logic (that belongs in
  `services/`)
- Be documented with a header comment describing what it renders and what
  data it expects
